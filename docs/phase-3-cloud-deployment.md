### 3c — AWS Setup (step by step)

```
Step 1: Create ECR repository
        aws ecr create-repository --repository-name industry-analyst

Step 2: Build + tag + push image
        docker build -t industry-analyst .
        docker tag industry-analyst <account>.dkr.ecr.<region>.amazonaws.com/industry-analyst:latest
        docker push ...

Step 3: Create S3 bucket
        aws s3 mb s3://industry-analyst-reports
        (enable versioning, block public access)

Step 4: Create IAM Role
        - Trust: App Runner / ECS Task
        - Policy: S3 PutObject + GetObject on the reports bucket (NO wildcard *)

Step 5: Store secrets
        aws secretsmanager create-secret --name prod/industry-analyst/openai
        aws secretsmanager create-secret --name prod/industry-analyst/serper

Step 6a (App Runner path):
        - Create App Runner service pointing to ECR image
        - Attach IAM role
        - Set env vars from Secrets Manager

Step 6b (ECS Fargate path):
        - Create Task Definition (reference ECR image + IAM role)
        - Create ECS Service with Fargate launch type
        - Add Application Load Balancer
        - Configure CloudWatch log group
```

### 3d — Real S3UploaderTool

Replace the local-file stub:
```python
# tools/s3_tool.py
import boto3

def upload_report(local_path: str, run_id: str) -> str:
    s3 = boto3.client("s3")   # uses IAM role — no keys in code
    key = f"reports/{run_id}/report.md"
    s3.upload_file(local_path, "industry-analyst-reports", key)
    url = s3.generate_presigned_url(
        "get_object",
        Params={"Bucket": "industry-analyst-reports", "Key": key},
        ExpiresIn=604800   # 7 days
    )
    return url
```

## Security Checklist

- [ ] No API keys in code or Dockerfile (Secrets Manager only)
- [ ] IAM role has minimum required permissions (no `*`)
- [ ] S3 bucket has public access blocked
- [ ] Presigned URLs expire in 7 days
- [ ] Docker image runs as non-root user
